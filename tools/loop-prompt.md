# StudyHub 전수 개선 루프 프롬프트

> 사용법: `/loop 20m [아래 내용 전체 붙여넣기]`
> 또는 파일 경로로: `/loop 20m @tools/loop-prompt.md`

## 실행 지시

너는 지금 StudyHub(F:\work\studyhub)의 **전수 개선 에이전트**다. 매 실행마다 아래 **8개 관점 중 다음 순번**을 골라 **한국어(ko/) 전체 페이지를 전수 감사**하고, 발견한 문제를 즉시 고쳐라.

### 필수 준비 단계 (매번 실행)
1. `tools/loop-state.json` 읽기 → `angleIndex` 확인 (없으면 0으로 시작)
2. 해당 인덱스의 관점(아래 목록) 선택
3. **`creative-brainstorming` 스킬 먼저 호출** → 해당 관점에서 StudyHub가 경쟁 사이트(네이버 지식백과, 나무위키, EBS, Khan Academy)보다 왜 부족한지, 어떻게 더 잘 할 수 있는지 아이디어 발굴
4. 시각적·레이아웃 개선이 포함되면 **`frontend-design` 스킬도 호출**
5. 감사 → 수정 → 전파 → 상태 저장 → 커밋

## 상세 실행 절차

### STEP 1 — 상태 파일 읽기
```
tools/loop-state.json 읽기
angleIndex = 파일의 angleIndex 값 (없으면 0)
currentAngle = 위 테이블의 해당 행
```

### STEP 2 — 브레인스토밍
`creative-brainstorming` 스킬을 호출하여 다음 질문에 답하라:
- StudyHub가 지금 **[현재 관점]** 측면에서 경쟁 사이트 대비 어디가 약한가?
- 한국 중·고등학생이 실제로 원하는 것은 무엇인가?
- 이 관점에서 즉시 적용 가능한 개선 아이디어 10가지 이상

시각/레이아웃 개선이 포함되면 `frontend-design` 스킬도 호출하여 구체적인 HTML/CSS 변경 방향 수립

### STEP 3 — 한국어 전수 감사
`F:\work\studyhub\ko\` 디렉토리의 **모든 HTML 파일**을 읽고 현재 관점 기준으로 문제점을 목록화:
- 파일명, 문제 유형, 심각도(critical/high/medium)
- 파일당 1분 이상 깊게 읽을 것

### STEP 4 — 한국어 즉시 수정
감사에서 발견된 문제를 **모두** ko/ 파일에 적용:
- critical → 반드시 이번 루프에서 수정
- high → 가능한 한 수정
- medium → 시간 남으면 수정
- 수정 시 기존 스타일(CSS 클래스, HTML 구조)을 최대한 재사용할 것

### STEP 5 — 다국어 전파
ko/ 변경 내용을 패턴에 따라 en/, ja/, zh/에도 동일하게 적용:
- 구조·레이아웃·컴포넌트 변경 → 모든 언어 동기화
- 한국어 전용 내용(국어 교과) → ja/japanese, zh/chinese, en/english 해당 페이지에 동등한 수준의 해당 언어 콘텐츠 추가
- 수학·과학·공통 영어 → 4개 언어 모두 동기화

### STEP 6 — 상태 저장
```json
{
  "angleIndex": (이전값 + 1) % 8,
  "lastRun": "ISO 날짜",
  "lastAngle": "관점 이름",
  "filesChanged": 수정된 파일 수,
  "summary": "이번 루프에서 한 일 한 줄 요약"
}
```

### STEP 7 — Git 커밋
```
git add ko/ en/ ja/ zh/
git commit -m "루프[각도명]: [한 일 요약]"
```

## 품질 기준 (매 루프마다 이 기준을 의식할 것)

### 콘텐츠
- [ ] 개념 설명이 **"왜"**로 시작하는가 (단순 정의 나열 금지)
- [ ] 예시가 학생의 실생활과 연결되는가
- [ ] 연습문제는 단순 계산 외에 **사고력 문제**가 포함되는가
- [ ] 오답 시 "어디서 틀렸는지" 알 수 있는 힌트가 있는가
- [ ] 암기팁·패턴·핵심 공식이 시각적으로 강조되는가

### 신뢰
- [ ] 수능 기출·교육과정 연계가 명시되어 있는가
- [ ] 틀리기 쉬운 부분(common mistakes)이 따로 표시되는가
- [ ] 수식·표가 정확한가 (MathJax 오류 없음)

### 리텐션
- [ ] 페이지 하단에 "다음에 공부할 것" 추천이 있는가
- [ ] 섹션 간 학습 경로(로드맵)가 보이는가
- [ ] "이걸 배우면 이게 가능해진다"는 동기 부여 문장이 있는가

### 차별화
- [ ] 이 페이지에만 있는 독자적인 설명·도표·비교표가 있는가
- [ ] 학생이 실수하기 쉬운 함정을 콕 짚어주는가
- [ ] 단순 정의보다 **시험에서 어떻게 나오는지** 안내가 있는가

## 주의사항
- **한국어 먼저** — ko/ 수정 완료 후 다국어 전파. 절대 순서 바꾸지 말 것
- 한 루프에 너무 많은 파일을 건드리기보다, **한 관점에서 완성도 높게** 수정할 것
- CSS는 `css/style.css`에 중앙화. 개별 파일에 inline style 추가 금지 (단, 부득이한 경우 제외)
- 기존 HTML 구조(`.concept-box`, `.tip`, `.example`, `.table-wrap` 등)를 최대한 활용
- 새 컴포넌트가 필요하면 `css/style.css`에 추가하고 전 언어에 일관 적용
